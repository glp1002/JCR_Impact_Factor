CLASES

En cuanto a las clases más o menos de forma general había pensado en hacer algo así:
 * Backend
    - Modelo:
        Clase para interactuar con la base de datos y obtener los datos necesarios.
        Clase para realizar las operaciones estadísticas sobre los datos filtrados según la selección del usuario.
    - Controlador:
        Clase para manejar las peticiones del usuario y enviar los resultados a la vista.
    - Vista:
        Clase para formatear la información de manera adecuada para su presentación al usuario (en caso de tener una interfaz gráfica)

 * Frontend
    - Vista:
        Clase para manejar la presentación de la información al usuario, puede contener clases específicas para cada componente o elemento visual que se presente en la interfaz.
        Clase para manejar eventos y acciones del usuario.
    - Controlador:
        Clase para manejar las solicitudes del usuario y enviarlas al backend.
        Clase para manejar la respuesta del backend y actualizar la vista en consecuencia.
